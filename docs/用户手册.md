# Smart Data Extractor 用户手册

本文档面向项目使用者，按当前代码里已经实现的功能说明安装、Web 使用、CLI 使用和常见问题。

## 1. 产品简介

Smart Data Extractor 是一个基于 LLM 的网页内容结构化提取与智能分析工具。

当前已经可用的能力包括：

- 单个网页结构化提取
- 批量 URL 提取与批次管理
- 页面类型分析与候选字段识别
- 单 URL 深度分析
- 多 URL 对比分析
- 任务历史、同 URL 轨迹、批量子任务查看
- 分析结果导出，以及任务详情页下载文本 / JSON

输出格式支持：`json`、`csv`、`sqlite`。

## 2. 环境准备

### 2.1 本地运行要求

- Python `>= 3.12`
- `uv` 包管理器
- 可访问的 OpenAI 兼容 LLM 接口

### 2.2 Docker 运行要求

- Docker `>= 24.0`
- Docker Compose `>= 2.0`

## 3. 安装与初始化

### 3.1 拉取代码并安装依赖

```bash
# 如果代码尚未在本机，请先从你的仓库托管平台克隆到本地
cd PersonalProject
uv sync
uv run playwright install chromium
```

说明：

- `uv sync` 用于安装项目依赖
- `uv run playwright install chromium` 用于安装动态抓取所需浏览器
- 如果你只使用静态抓取，理论上不一定每次都需要 Playwright，但建议仍安装完整环境

### 3.2 配置 API 凭证

推荐优先使用环境变量。

Linux / macOS：

```bash
export SMART_EXTRACTOR_API_KEY=your-api-key
export SMART_EXTRACTOR_BASE_URL=https://api.openai.com/v1
export SMART_EXTRACTOR_MODEL=gpt-4o-mini
export SMART_EXTRACTOR_WEB_API_TOKEN=your-web-api-token
```

Windows PowerShell：

```powershell
$env:SMART_EXTRACTOR_API_KEY="your-api-key"
$env:SMART_EXTRACTOR_BASE_URL="https://api.openai.com/v1"
$env:SMART_EXTRACTOR_MODEL="gpt-4o-mini"
$env:SMART_EXTRACTOR_WEB_API_TOKEN="your-web-api-token"
```

补充说明：

- `SMART_EXTRACTOR_API_KEY`：必须项，没有它就无法提交新任务
- `SMART_EXTRACTOR_WEB_API_TOKEN`：Web API 鉴权用；如果配置了，前端访问 `/api/*` 时必须带 Token
- `config/default.yaml` 只保留安全默认值，不再放真实密钥
- Web 首页也支持直接编辑基础 LLM 配置并写回 `config/local.yaml`

### 3.3 验证 API 连通性

```bash
uv run smart-extractor test-api
```

## 4. CLI 使用指南

### 4.1 查看帮助

```bash
uv run smart-extractor --help
```

### 4.2 单页提取 `extract`

自动模式示例：

```bash
uv run smart-extractor extract "https://example.com/article"
```

固定 Schema 示例：

```bash
uv run smart-extractor extract "https://example.com/article" --schema news
```

常用参数：

- `--schema/-s`：Schema 名称，默认 `auto`
- `--format/-f`：输出格式（`json` / `csv` / `sqlite`）
- `--collection/-c`：输出集合名
- `--static`：使用静态抓取
- `--selector`：仅提取指定 CSS 区域
- `--verbose/-v`：显示详细日志
- `--fields`：仅在 `auto` 模式下手动指定字段

### 4.3 批量提取 `batch`

先准备 `urls.txt`：

```text
https://example.com/a1
https://example.com/a2
```

执行：

```bash
uv run smart-extractor batch urls.txt --schema news --format json --workers 5
```

常用参数：

- `--workers/-w`：并发数；传 `0` 时使用配置默认值
- `--static`：使用静态抓取模式

### 4.4 其它命令

- `schemas`：列出当前可用 Schema
- `config`：查看当前配置（密钥会脱敏）
- `test-api`：测试 LLM 连通性
- `web`：启动 Web 仪表盘
- `runtime`：查看任务、监控与成本概览
- `monitors`：列出已保存监控与通知策略
- `templates`：列出已保存模板
- `template-from-task`：从成功任务沉淀模板
- `smoke-sites`：执行真实站点烟雾验证

## 5. Web 仪表盘使用

### 5.1 启动服务

```bash
uv run smart-extractor web
```

可选参数：

- `--host/-H`：监听地址，默认 `127.0.0.1`
- `--port/-p`：监听端口，默认 `8000`
- `--reload`：开发模式自动重载

访问地址：`http://127.0.0.1:8000`

### 5.2 首页运行状态

默认情况下，Web 会尽量先启动成功，并把当前状态显示在首页：

- 未配置 `SMART_EXTRACTOR_API_KEY` 时，可浏览界面，但不能提交任务
- 未配置 `SMART_EXTRACTOR_WEB_API_TOKEN` 时，接口会以无鉴权模式运行，仅建议本机调试使用
- 可通过 `SMART_EXTRACTOR_STARTUP_CHECK_ENABLED=true` 开启启动自检

首页当前主要模块包括：

- 运行状态
- API 鉴权
- 基础配置
- 任务总览与成功率
- 重复监控候选
- 开始任务
- 批量提交
- 智能分析
- 任务进展

### 5.3 基础配置

首页“基础配置”面板可以直接编辑：

- `API KEY`
- `BASE URL`
- `MODEL`
- `TEMPERATURE`

保存后会写入 `config/local.yaml`。

注意：

- 如果相关环境变量已经设置，环境变量仍会覆盖文件值
- 页面会提示哪些字段当前正被环境变量接管

### 5.3.1 成本与收益概览

首页/接口会聚合以下信息：

- LLM 调用次数
- Prompt / Completion Tokens
- 估算成本（USD）
- 站点记忆复用带来的估算节省

说明：

- 当前成本为基于模型价格表的估算值，适合做趋势对比与收益表达
- 未命中定价表的模型会显示为 `0`

### 5.4 单任务提取

在“开始任务”中：

1. 输入目标 URL
2. 选择存储格式：`json` / `csv` / `sqlite`
3. 选择抓取模式：动态抓取或静态抓取
4. 直接“提交任务”，或先“分析页面”再勾选字段

分析页面后，系统会展示：

- 页面类型
- 候选字段
- 页面预览

提交成功后，还可以：

- 从成功任务直接生成模板草案
- 导出 Word / Excel / Markdown / JSON
- 把任务沉淀成后续可复用模板

当前真实流程是：

`提交任务 -> 到任务进展查看状态 -> 进入详情页查看 / 复制 / 下载结果`

不是“提交后自动弹出下载文件”。

### 5.5 批量提交

在“批量提交”中：

- 每行输入一个 URL
- 页面会自动统计有效数量、重复项、无效行
- 页面会按域名和路径生成关系预览
- 支持“新建批次”和“续接已有批次”

批量提交后，系统会创建一个父任务，并自动生成多个子任务。

在任务详情页里可以看到：

- 批量进度
- 成功 / 失败统计
- 子任务明细

### 5.6 智能分析

“智能分析”不是普通字段抽取，而是把网页内容和你的目标一起交给系统，生成更偏决策型的结论。

当前支持两种模式。

#### 单 URL 深度分析

适合：

- 快速总结页面
- 结合身份和目标做判断
- 生成风险提示或行动建议

流程：

1. 输入 URL
2. 点击“预分析 URL”
3. 填写分析目标、身份、优先级、限制条件、补充说明
4. 点击“生成分析”

#### 多 URL 对比分析

适合：

- 多商品比较
- 多岗位比较
- 多方案比较

流程：

1. 输入多个 URL
2. 点击“预览对比对象”
3. 填写重点维度、必须满足、淘汰条件、补充说明
4. 点击“生成分析”

#### 分析结果支持的操作

当前支持：

- 导出汇报文本
- 导出 Markdown
- 导出 CSV
- 导出 JSON

### 5.7 任务进展与详情页

“任务进展”页会显示最近任务，包括：

- 任务编号
- 批次
- URL
- 模式
- 状态
- 质量分
- 耗时
- 创建时间
- 详情入口

任务详情页当前支持：

- 查看实时进度条
- 查看同 URL 历史轨迹
- 查看批量子任务
- 查看错误信息
- 查看润色结果
- 查看原始 JSON
- 复制润色结果
- 下载润色结果
- 下载原始 JSON

如果后端启用了 API Token 鉴权，详情页左侧也需要填写 Token，页面才能持续刷新。

## 6. 输出与目录说明

- `output/`：提取结果输出目录
- `logs/`：运行日志目录
- `config/schemas/`：自定义 Schema 目录

补充：

- Web 任务状态和历史记录会保存在输出目录下的 SQLite 数据库中
- 批量去重缓存位于 `output/.url_dedup_cache`

## 7. 自定义 Schema

在 `config/schemas/` 下创建 YAML 文件即可扩展固定字段模板。

示例：

```yaml
name: "CustomModel"
description: "自定义提取模板"
fields:
  title:
    type: str
    description: "标题"
    required: true
  score:
    type: float
    description: "评分"
    default: 0
```

创建后可执行：

```bash
uv run smart-extractor schemas
```

确认是否已加载。

## 9. 常见问题

### 9.1 `test-api` 失败

排查顺序：

1. 确认 `SMART_EXTRACTOR_API_KEY` 已正确设置
2. 确认 `SMART_EXTRACTOR_BASE_URL` 可访问
3. 确认模型名可用（`SMART_EXTRACTOR_MODEL`）
4. 查看日志目录中的错误日志

### 9.2 页面能打开，但不能提交任务

最常见原因：

- 没有配置 `SMART_EXTRACTOR_API_KEY`
- 当前基础配置写入了文件，但又被环境变量覆盖
- API Token 不正确，导致 `/api/*` 请求失败

建议先查看首页“运行状态”和“基础配置”面板。

### 9.3 Web 接口提示 401

说明后端启用了 `SMART_EXTRACTOR_WEB_API_TOKEN`。

处理方式：

- 在首页“API 鉴权”输入 Token
- 在任务详情页左侧也输入相同 Token

### 9.4 有些页面抓不到真实内容

可能原因：

- 目标站点返回安全验证页
- 页面还停留在加载状态
- 页面被站点风控，只返回了前端壳页
- 网页清洗后文本为空

### 9.5 动态抓取和静态抓取怎么选

- 页面依赖前端渲染：优先动态抓取
- 页面结构简单、只想快速试跑：优先静态抓取

### 9.6 批量任务重复抓取

系统内置 URL 去重缓存：`output/.url_dedup_cache`。

如需重新抓取全部 URL，可在确认后清理该缓存文件。

## 10. 安全建议

- 不要把真实 API Key 提交到代码仓库
- 优先使用环境变量或密钥管理服务注入凭证
- 若未配置 `SMART_EXTRACTOR_WEB_API_TOKEN`，Web API 会处于无鉴权模式，仅建议本机调试使用

### 5.6 监控通知降噪

监控 profile 现在支持以下抑噪策略：

- `digest_only`：仅进入 Digest，不做实时通知
- `quiet_hours_enabled` + `quiet_hours_start/end`：静默时段
- `notification_cooldown_minutes`：通知冷却时间
- `min_change_count`：最小变化字段数阈值
- `min_change_ratio`：最小变化占比阈值

说明：

- `error` 告警默认可绕过部分抑噪规则，避免关键失败被静默
- 这些规则适合控制高频监控的提醒疲劳
